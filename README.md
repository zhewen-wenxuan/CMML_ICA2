# CMML ICA2: scDesign3 trajectory inference benchmark

This repository contains the code and generated outputs for **CMML3 ICA2 Miniproject 3: Simulating single-cell RNA-seq data with scDesign3**.

The project uses **scDesign3** to simulate pseudotime-dependent single-cell RNA-seq data and benchmarks three R-based trajectory inference methods:

- Slingshot
- Monocle3
- TSCAN

The aim is to use scDesign3 as a controlled simulator, rather than to optimise scDesign3 itself. The synthetic datasets have known pseudotime ground truth, allowing different trajectory inference methods to be compared objectively.

---

## Project summary

The analysis follows this workflow:

```text
Reference scRNA-seq data with pseudotime
→ scDesign3 baseline simulation
→ three simulated scenarios
→ Slingshot / Monocle3 / TSCAN
→ pseudotime recovery metrics
→ figures and output files
```

Three simulated scenarios were generated:

| Scenario | Description |
|---|---|
| S1 baseline | Direct scDesign3 synthetic dataset |
| S2 low-depth | Count downsampling to mimic lower sequencing depth and increased sparsity |
| S3 imbalanced-middle | Downsampling of middle-pseudotime cells to mimic incomplete sampling of intermediate states |

## Data

The reference dataset is the scDesign3 quickstart pancreatic endocrinogenesis example dataset:

```r
example_sce <- readRDS(url("https://figshare.com/ndownloader/files/40581992"))
```

The full dataset contains 1000 highly variable genes and 2087 cells with pseudotime annotation. Following the course quickstart, the first 100 genes were used to reduce computational cost:

```r
example_sce <- example_sce[1:100, ]
```

The data are downloaded automatically by the R script. If automatic download fails, the same file can be downloaded manually from:

```text
https://figshare.com/ndownloader/files/40581992
```

## How to run

Open R or RStudio, set the working directory to the repository root, and run:

```r
source("run_CMML3_ICA2_scDesign3_trajectory.R")
```

The script will:

1. Load the required R packages.
2. Load or download the reference scRNA-seq dataset.
3. Run scDesign3 simulation.
4. Generate S1 baseline, S2 low-depth and S3 imbalanced-middle scenarios.
5. Run Slingshot, Monocle3 and TSCAN.
6. Calculate pseudotime recovery metrics.
7. Save figures, metrics and session information.

## Evaluation metrics

Trajectory inference performance was evaluated by comparing inferred pseudotime with true pseudotime.

The main metrics were:

| Metric | Meaning |
|---|---|
| Spearman correlation | Rank agreement between true and inferred pseudotime |
| Kendall correlation | Additional rank-based ordering agreement |
| Scaled RMSE | Numerical error after scaling pseudotime to 0-1 |
| Adjusted Rand index | Recovery of early, middle and late pseudotime states |

If inferred pseudotime was reversed, its direction was corrected before metric calculation. This prevents methods from being penalised for recovering the correct trajectory in the opposite direction.

## Result summary

The baseline scDesign3 synthetic dataset preserved the main cell-level QC properties and continuous pseudotime structure of the reference dataset.

In the baseline scenario, Slingshot, Monocle3 and TSCAN all recovered pseudotime well. Under the low-depth and imbalanced-middle scenarios, method-specific differences became clearer. Slingshot and TSCAN remained relatively robust, while Monocle3 was more sensitive to depletion of intermediate-pseudotime cells in this simplified benchmark.

This project is a focused benchmark for a single-lineage trajectory, not a universal ranking of all trajectory inference methods.
